# 감정-색상 팔레트 설계 가이드

## 개요
SentiVision의 색상-감정 매핑은 단순한 1:1 매핑이 아니라, **색상의 미세한 차이를 통해 감정의 세부 뉘앙스를 표현**하는 것을 목표로 합니다. 본 문서는 라벨의 정의, 색상 범위, 세분화 원칙을 명시합니다.

## 핵심 원칙

1. **감정 라벨은 유지**: 범주를 줄이지 않고, 각 감정의 색적 표현력을 높입니다.
2. **색상 중심부(representative RGB)**: 각 감정마다 대표 색상을 하나 정하고, 주변부는 변형 샘플로 채웁니다.
3. **최소 샘플 수**: 각 감정마다 최소 3개 이상의 RGB 샘플을 보유합니다.
4. **경계 분명성**: 같은 감정의 색들이 너무 분산되지 않으면서도, 다른 감정과는 구분 가능해야 합니다.

## 사용 규칙

### 모델 학습 시
- 각 감정마다 대표RGB를 중심으로 ±5~20의 RGB 변형을 추가합니다.
- 같은 감정의 색들은 색상공간(HSV/LAB)에서 클러스터를 이루어야 합니다.
- 다른 감정과의 경계는 유클리드 거리 기준 30 이상 떨어져 있는 것을 권장합니다.

### 사용자 피드백 추가 시
- 새 샘플은 기존 감정들과 RGB 충돌이 없는지 먼저 확인합니다.
- 새 감정은 기존 라벨과 색상상 겹침이 없을 때만 추가합니다.
- 매월 1회 이상 감정별 샘플 분포를 재확인합니다.

### 라벨 업데이트 정책
- 대표RGB 변경: 전체 샘플의 평균이 ±10 이상 이동했을 때
- 감정 라벨 추가: 기존 라벨과 색상상 구분 가능하고, 최소 5개 샘플 확보 시
- 감정 라벨 제거: 지원되지 않음 (기존 라벨은 영구 유지)
