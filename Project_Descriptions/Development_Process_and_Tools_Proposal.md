# 개발 과정 및 필요 도구 제안서

작성일: 2026-03-20  
문서 버전: v1.1
프로젝트: SentiVision

## 1. 문서 목적
- 현재 저장소의 실제 구현 범위(파이썬 CLI 분석 스크립트)에 맞는 개발 과정을 정리한다.
- 단계별로 필요한 도구와 운영 규칙을 제안한다.
- 향후 API/UI 확장 시 참고 가능한 기준선을 제공한다.

## 2. 현재 프로젝트 기준선

### 2-1. 구현 형태
- 실행 방식: 로컬 CLI (`python test/main_.py`)
- 핵심 기능: 이미지 주요 색상 추출 + 감정 예측 + 사용자 피드백 반영
- 산출물: 시각화 PNG 3종, CSV 갱신(정정 입력 시)

### 2-2. 핵심 파일
- 실행 스크립트: `test/main_.py`
- 주 데이터셋: `test/color_emotion_labeled_updated.csv`
- 보조 데이터셋: `test/colorassociations_warmth - colorwarmth.csv`
- 출력 폴더: `test/outputs/`

## 3. 개발 과정 제안 (현행화)

### 단계 1. 요구사항/문서 정합화
- 목표: 문서와 실제 구현 범위를 일치시킨다.
- 작업:
  - PRD/WBS/User Journey/Wireframe 문서를 CLI 기준으로 유지
  - README와 Project_Descriptions 파일 목록 동기화
- 완료 기준:
  - 문서 내 경로/용어가 저장소와 모두 일치

### 단계 2. 데이터 품질 관리
- 목표: 학습/예측 데이터의 신뢰도를 유지한다.
- 작업:
  - CSV 로딩 가능성 점검
  - 중복/결측 정책 명문화
  - 라이선스/서드파티 고지 유지
- 완료 기준:
  - 데이터셋 로드 실패 없이 분석 실행 가능

### 단계 3. 분석 파이프라인 안정화
- 목표: 입력 이미지에 대해 안정적으로 결과를 생성한다.
- 작업:
  - KNN 학습/예측 유지
  - 현저성 추출 + KMeans 주요색 추출 검증
  - 경로 오류/현저영역 미검출 예외 동작 확인
- 완료 기준:
  - 정상 입력 시 예측 결과와 시각화 파일 생성

### 단계 4. 피드백 루프 개선
- 목표: 사용자 정정이 데이터셋 개선으로 이어지게 한다.
- 작업:
  - 입력 규칙(Enter/yes/y/예) 유지
  - 정정 입력 시 CSV 반영 및 중복 제거 보장
- 완료 기준:
  - 정정 입력 반영, 수락 입력 미반영이 일관되게 동작

### 단계 5. 운영 지표 관리
- 목표: 저장소 운영 관점의 품질 지표를 지속 추적한다.
- 작업:
  - GitHub Actions DORA 워크플로 점검
  - incident 라벨 규칙 준수
- 완료 기준:
  - 워크플로 수동 실행 및 요약 확인 가능

## 4. 필요 도구 제안 (Tooling)

### 4-1. 필수 도구
- Python 3.10+
- 가상환경: `venv`
- 패키지: pandas, numpy, scikit-learn, opencv-python, matplotlib

### 4-2. 개발/형상관리
- Git + GitHub
- 브랜치/커밋 규칙 유지 (`feat`, `fix`, `docs` 등)

### 4-3. 품질 관리
- 수동 실행 체크리스트
- 선택: pytest 기반 스크립트 동작 검증 코드 추가

### 4-4. 자동화/운영
- GitHub Actions (DORA 메트릭 워크플로)
- README 운영 규칙(incident 라벨, 수동 실행 절차) 준수
